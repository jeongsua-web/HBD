## 1. 개요 (Overview)

수아의 생일파티를 위한 **초대장형 웹사이트**.
초대받은 친구들이 파티 정보를 확인하고, 닉네임을 만들어 공지 확인 인증 댓글을 남기는
가볍고 감성적인 이벤트 페이지다.

- **목적**: 파티 정보 전달 + 참석자 참여(댓글) 유도 + 분위기 띄우기
- **컨셉**: Y2K / Barbie 핑크 무드 ("On June 29th, we wear PINK!")
- **호스트**: 수아 (Sua)
- **운영 기간**: 즉시 ~ **2026-06-30** (파티 종료 다음 날까지)
- **파티 일시**: **2026-06-29 (월) 17:00** (호스트 15:00부터 대기)

## 2. 목표 (Goals) & 비목표 (Non-goals)

### Goals
- 모바일에서 빠르게 열리는 가벼운 정적 페이지
- D-DAY 실시간 카운트다운으로 기대감 형성
- 닉네임 기반의 친근한 참여형 댓글 피드
- 별도 설치·로그인 없이 링크 공유만으로 접근
- 수아의 위시리스트로 파티 참여자의 선물 고민 종결 (파티원이 찜하면 표시가 됨)

### Non-goals
- 회원가입 / 비밀번호 인증 시스템
- 결제·선물펀딩 기능
- 관리자(어드민) 대시보드 (호스트는 댓글 수집 도구로 대체)
- 운영 종료(6/30) 이후의 지속적인 유지보수

### 우선순위 (MoSCoW)

> 목적: **6/29 마감**에 시간이 부족할 때 무엇을 자르고 무엇을 사수할지 미리 합의.

| 등급 | 항목 | 비고 |
|------|------|------|
| **Must** (필수) | F1 타이머 · F2 공지 보드 · F3 지도 링크 · F4 닉네임 | 정적만으로 동작, 페이지의 뼈대 |
| **Should** (핵심) | F5 실시간 댓글 · F6 위시리스트+찜 | 호스트가 가장 원하는 기능. 우선순위 높음 |
| **Could** (여유 시) | BGM · 입장 애니메이션 등 재미 요소 | 시간 남을 때만 |
| **Won't** (이번엔 제외) | 어드민 대시보드 · 결제/펀딩 · 푸시 알림 | Non-goals와 동일 |

> 막판 컷 대상은 **Could** 우선, 그다음 Should 내에서 F6 → F5 순으로 축소 검토.

## 4. 핵심 기능 (Features)

### F1. D-DAY 실시간 타이머
- 목표 시각(2026-06-29 17:00)까지 **일·시·분·초** 1초 단위 갱신
- 파티 시작 시각 도달 시 → `IT'S SHOW TIME! 💖` 등 축하 메시지로 전환
- **수용 기준**
  - 페이지 진입 즉시 카운트다운 표시
  - 시간 경과에 따라 매초 정확히 감소
  - 목표 시각 이후 음수 표시 없이 종료 문구로 교체

### F2. 공지사항 보드 (일시 · 장소 · 드레스코드)
- 정적 공지 영역에 다음 항목 노출
  - 일시 / 장소(주소) / 드레스코드 / 사전 미션 / 매너 타임 등 안내
- **수용 기준**
  - 모든 공지가 스크롤 없이 또는 짧은 스크롤로 읽힘
  - 강조 항목(드레스코드 등) 시각적으로 구분

### F3. 네이버 지도 길찾기 아웃링크
- 파티 장소 주소 노출 + **"네이버 지도로 길찾기"** 버튼
- 버튼 클릭 시 새 탭(`target="_blank"`)으로 네이버 지도 열림
- **수용 기준**
  - 정확한 장소 좌표/검색어로 연결 (TBD: 정확한 지도 URL 확정 필요)
  - 모바일에서 네이버 지도 앱/웹으로 정상 연결

### F4. 최초 접속 닉네임 설정 팝업
- 첫 방문 시 모달 팝업으로 닉네임 입력 요청
- 입력값은 `localStorage`에 저장 → 재방문 시 팝업 생략
- **수용 기준**
  - 빈 닉네임 제출 방지 (검증 메시지)
  - 저장 후 페이지 타이틀/인사말에 닉네임 반영
  - 재접속 시 모달 미노출

### F5. 닉네임 기반 공지 확인 댓글 피드
- 저장된 닉네임으로 "공지 확인" 한마디 댓글 작성
- 작성한 댓글이 **모든 참여자에게 공유**되는 실시간 피드에 표시
- **수용 기준**
  - 닉네임 없이 댓글 작성 불가
  - XSS 방지를 위한 입력값 이스케이프 처리
  - 작성 즉시 본인 화면 반영 + 다른 접속자 화면에도 실시간 반영

### F6. 수아 위시리스트 & 찜(Claim) 기능 ✨
- 수아가 받고 싶은 선물 목록을 카드로 노출 (이미지/이름/가격/구매링크)
- 참여자가 선물을 **"찜"** 하면 모두에게 **누가 찜했는지(닉네임)** 실시간 표시 → 선물 중복 방지
- **규칙**
  - 한 선물은 **1명만 찜 가능** (이미 찜된 항목은 잠금/예약됨 표시)
  - **1인당 최대 3개**까지 찜 가능
  - 찜한 사람의 **닉네임 공개** (닉네임 자체가 익명이므로 무방)
  - 본인이 찜한 항목은 **찜 취소** 가능
- **수용 기준**
  - 닉네임 없이 찜 불가
  - 이미 찜된 선물에는 찜 버튼 비활성화 + 찜한 닉네임 노출
  - 4개째 찜 시도 시 "최대 3개까지" 안내 후 차단
  - 다른 접속자의 찜/취소가 실시간 반영

## 5. 백엔드 & 데이터 모델 (Backend)

실시간 공유 댓글 피드(F5)와 위시리스트 찜(F6)은 모든 참여자가 같은 데이터를 봐야 하므로
공유 백엔드가 필요하다. **→ Supabase 채택 (결정 완료, 2026-06-05)**

**채택 사유**
- 호스트가 **다른 프로젝트에서 이미 Supabase 사용 중** → 셋업·SDK·인증 패턴 재사용, 학습 비용 0
- 위시리스트 "선물 ↔ 찜한 사람"은 **관계형 데이터** → Postgres가 자연스럽게 모델링
- Realtime 구독으로 댓글/찜 상태 실시간 반영
- 무료 티어로 파티 규모 트래픽 충분

### 테이블 스키마

```sql
-- 공지 확인 댓글
comments (
  id          uuid primary key default gen_random_uuid(),
  nickname    text not null,
  message     text not null,
  created_at  timestamptz default now()
)

-- 위시리스트 선물 항목 (호스트가 사전 입력)
wishlist_items (
  id          uuid primary key default gen_random_uuid(),
  title       text not null,
  image_url   text,
  link_url    text,          -- 구매 아웃링크
  price       text,
  sort_order  int default 0
)

-- 찜 (선물 1개당 1행, 즉 1명 독점)
wishlist_claims (
  id          uuid primary key default gen_random_uuid(),
  item_id     uuid not null references wishlist_items(id) unique,  -- 항목당 1명
  nickname    text not null,
  created_at  timestamptz default now()
)
```

**규칙 적용 방식**
- 선물 1개당 1명 → `wishlist_claims.item_id` 에 `UNIQUE` 제약
- 1인당 최대 3개 → 클라이언트 검증 + (선택) DB 트리거/RPC로 서버단 보강
- 닉네임 공개 → `claims.nickname` 그대로 노출

### 보안 (RLS)
- 회원 인증이 없으므로 `anon` 키로 접근. **Row Level Security** 정책으로 노출 통제
  - `comments` / `wishlist_claims`: `INSERT` 허용, `SELECT` 허용
  - `DELETE`(찜 취소): 클라이언트가 보유한 닉네임과 일치하는 행만 (간이 검증)
  - `wishlist_items`: `SELECT`만 허용 (등록은 호스트가 콘솔에서)
- ⚠️ 닉네임 기반이라 완벽한 위조 방지는 불가 — 파티용 신뢰 모델로 충분하다고 판단

## 6. 기술 스택 (Tech)

- **프론트엔드**: 순수 HTML / CSS / Vanilla JS (단일 `index.html`)
- **디자인 시스템**: `y2k-design-system.html` (Y2K 맥시멀 핑크 토큰)
- **호스팅**: 정적 호스팅 (Netlify 등) — 무료
- **백엔드/DB**: **Supabase** (Postgres + Realtime), CDN `@supabase/supabase-js`
- **로컬 상태**: 닉네임은 `localStorage` 유지 (간이 세션)
- **폰트**: Bagel Fat One, Chicle, Pixelify Sans, VT323, Quicksand

> Supabase URL / anon key는 클라이언트에 노출되는 공개키이며, 보호는 5장 RLS로 처리.
